Reflection

Looking back to move forward

When I first saw the assignment brief — build a personal portfolio website using Git, Quarto, and GitHub Pages — my immediate reaction was panic. I had never used any of these tools before. Coming from a background in traditional banking risk management, my technical work had always lived in SAS, Excel, and internal systems. Writing code to build a website felt like a completely different world.

Note My starting point

Before this module I had never used Git, Quarto, or VS Code. I associated portfolio websites with designers and computer science students — not with actuaries or risk analysts.

What I discovered

The biggest surprise was realising that building a portfolio with code is not that different from creating an Office document — you just need to be careful with the commands and the structure. Once I understood the logic, it stopped feeling scary and started feeling like a puzzle I wanted to solve.

What I didn’t expect was how much I would enjoy it. Once I got started, I found myself wanting to keep going — discovering what else I could do with Quarto, how to customise the design, how to make the Python code actually run inside the page. That curiosity kept me moving forward.

Challenges I overcame

The hardest part was getting started. I spent a lot of time hesitating before actually opening VS Code and writing my first command. The project felt overwhelming from a distance. But once I took the first step, the rest followed naturally.

Some specific technical challenges I faced:

Challenge What I learned
Git from scratch Every commit tells the story of your project
SCSS layer structure scss:defaults and scss:rules must be explicit
Quarto configuration Small YAML errors can break the entire site
Python environment Dependencies must be installed incrementally

If I were starting again, I would jump in earlier and build the portfolio little by little from day one — getting my hands dirty from the beginning instead of waiting until I felt “ready.” I’ve learned that with new tools, you only feel ready once you’ve already started.

Connecting to my professional future

This reflection is not just academic. I want to work at a fintech — somewhere like Revolut or Nubank — and one thing that distinguishes those companies from traditional banks is that their risk and analytics teams work in modern, collaborative, open environments. They use Git, they build in public, they share their work.

Note Key insight

Learning Git, Quarto, and web publishing has given me a competitive edge. These are not the most common skills in risk management — and that’s exactly why they matter. Being a risk analyst who can also build, publish, and communicate technical work professionally is a differentiator.

The fear of doing something new will always be there. But this module taught me that stepping outside your comfort zone — even when it feels uncomfortable — is where the real growth happens. I came in thinking this was a task for designers and developers. I’m leaving knowing it’s a skill every data professional should have.

What comes next

  • Start new projects earlier and build incrementally from day one
  • Continue developing this portfolio beyond this module
  • Deepen my Python skills in survival analysis and credit loss forecasting
  • Apply for risk analytics roles at fintechs in Ireland and the UK
